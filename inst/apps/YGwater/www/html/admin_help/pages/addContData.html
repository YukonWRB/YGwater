Add continuous data

Use this page to append continuous measurements to an existing AquaCache timeseries. This tool uploads values for records that already exist in continuous.timeseries. It does not create new monitoring locations and it does not create new timeseries records directly in the upload workflow.

Before you start

Select the target timeseries first. The list of available timeseries comes from AquaCache timeseries metadata and includes the location, parameter, unit, media type, aggregation type, and recording rate. Each timeseries is associated with an existing location through location_id, and may also be associated with a sub-location through sub_location_id. If the timeseries you need is not listed, create it first from Add / edit timeseries. If the monitoring location or sub-location does not exist yet, create that first from the location tools, then create the timeseries, then return here to upload data.

Appending data

Continuous data uploads are intended to add values to an existing timeseries. Confirm that the selected timeseries represents the same location, sub-location, parameter, unit, media, aggregation, and recording interval as the file or manual values you are about to upload. Uploading to the wrong timeseries can make later review, correction, publication, and data downloads misleading.

Units and timestamps

The page shows the database unit for the selected timeseries. Convert incoming values to the database unit before upload, or use the available unit conversion tools where appropriate. Check timestamps carefully, including timezone and daylight saving transitions. The uploaded date/time values should represent the measurement times for the selected timeseries, not the file creation time or logger download time.

Review before committing

Use the plotting and review tools before committing data. Compare the new values against recent values and daily summaries where available. Confirm grades, approvals, qualifiers, owners, contributors, and any overwrite choices before upload. Only delete values that have no possible later use, such as obvious pre-deployment or post-recovery noise; values that may be useful later should usually be uploaded and handled with corrections, grades, approvals, or qualifiers.

Timeseries and instrument deployments

A timeseries can be associated with an instrument deployment. This association is useful when the measurements were produced by a deployed logger, sensor, or instrument, because it preserves the operational context for the data: which instrument was in the field, where it was deployed, when the deployment started and ended, and how the data source relates to maintenance, calibration, recovery, and telemetry records.

Deployment associations help later users understand why values changed, which sensor produced a record, which deployment a gap or shift belongs to, and whether data should be interpreted with information from checks, calibrations, repairs, or deployment notes. They also help separate long-term station timeseries from the physical instruments that may be swapped, repaired, recalibrated, or redeployed over time.

Use Deploy / recover instruments to create and maintain deployment records in public.locations_metadata_instruments. Direct deployment-to-timeseries links are stored in public.locations_metadata_instrument_timeseries. Use Add / edit timeseries or the deployment workflow to associate timeseries with active deployments. Some telemetry workflows link timeseries at the connection-signal level through public.locations_metadata_instrument_connection_signals.timeseries_id; when that is the case, update the link in the Acquisition / telemetry tools rather than forcing a direct deployment-timeseries association.

If something is missing

If the target location, sub-location, timeseries, instrument, sensor, deployment, or telemetry connection is missing, add the missing metadata before uploading values. Continuous measurements are much easier to maintain when the database structure is correct before data are appended.

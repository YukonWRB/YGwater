Add / edit timeseries

Use this page to create or update continuous-data timeseries metadata in AquaCache. A timeseries defines what a continuous measurement means: where it is measured, what parameter it measures, the media and matrix state, how values are aggregated, who owns the data, who can see it, and how it may be downloaded or corrected.

Add or modify mode

Choose Add new timeseries to create a new record in continuous.timeseries. Choose Modify existing timeseries to select a timeseries from the table and update its metadata. In modify mode, review the selected record carefully before changing fields that affect existing data interpretation.

Location and sub-location

Select the parent location first. If the data are tied to a specific point within that location, select the sub-location as well. Missing locations and sub-locations must be created in the location tools before the timeseries is created.

Timezone for daily aggregation

The timezone field controls how AquaCache groups raw timestamps into daily statistics. It should normally match the standard local timezone for the site. This setting does not change raw timestamps; it affects daily summary boundaries. Changing it on an existing timeseries may require recalculating statistics from the beginning of the record.

Elevation or depth

Use elevation or depth when measurement height matters for interpretation, such as air temperature height, wind sensor height, water level reference point, or sensor depth. Existing values for the selected location or sub-location are shown; type a new numeric value if needed.

Parameter, media, and matrix state

Select the parameter, media, and matrix state together. Matrix states are only available when the parameter has units configured for that state. If the expected matrix state is missing, update the parameter reference data first, then reload this module.

Aggregation type and record rate

The aggregation type describes how values represent time, such as instantaneous values or means over an interval. The record rate should describe the recent collection interval, such as 5 minutes, 1 hour, or 1 day. Use a reasonably accurate rate because it affects expectations for gaps, summaries, and historical range calculations.

Sensor priority

Sensor priority distinguishes primary, secondary, and tertiary sources for the same parameter or site. Use primary for the preferred operational record and lower priorities for backups or alternate sensors.

Owner, sharing agreement, and access

The default owner is applied to uploaded data when no owner is specified at upload time. If the owner has an associated data sharing agreement, select the relevant agreement so compliance can be tracked. Use Share with groups to control which database roles can see the timeseries; public_reader makes it broadly visible.

Instrument association

Use the optional instrument association when the timeseries is produced by a currently deployed instrument. The list is filtered to deployments for the selected location and sub-location. Linking the timeseries to the deployment preserves which instrument produced the measurements and connects the data to deployment, recovery, calibration, maintenance, and telemetry context.

If a deployment uses connection-signal metadata, manage the timeseries link under the Acquisition / telemetry tools. Those links are stored at the signal level and should not be replaced by a direct deployment-timeseries link.

Auto-download options

Use a source function when AquaCache should fetch data automatically from an external source. Source functions are AquaCache functions whose names begin with download. Leave this section blank for manually uploaded data or data loaded through other workflows. Source function arguments must be entered as key-value pairs that can be converted to JSON.

Automatic corrections and filters

When creating a new timeseries, you can add broad default corrections such as trim or linear offset corrections. These corrections are inserted with broad UTC bounds. Use them only when the correction should apply generally to the timeseries. Existing corrections are shown in modify mode to help avoid duplicates.

Before saving

Confirm the location, sub-location, parameter, media, matrix state, unit, aggregation type, record rate, timezone, owner, sharing agreement, access groups, and instrument association. After creating the timeseries, use Add continuous data to append measurements.

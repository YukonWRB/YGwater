Add / modify locations

Use this page to create a new monitoring location or update metadata for an existing location in AquaCache. Locations are the parent records for sub-locations, timeseries, samples, documents, images, deployments, and many other records, so it is worth getting the location metadata right before adding related data.

Add or modify mode

Choose Add new location to create a new record. Choose Modify existing location to select a location from the table and update its fields. In modify mode, selecting a row loads the current location details into the form and changes the main button to Update location.

Location codes

Every location needs a unique location code. For non-Water Survey of Canada locations, use the auto-generate button unless you have a specific code that should be used and that code is based on National Hydro Network logic. Auto-generated codes reduce duplicates and keep codes consistent across the database.

If the location is a Water Survey of Canada station and HYDAT is available, enter the WSC code and use the HYDAT auto-fill option to populate available station details. Review all auto-filled fields before saving.

Names and aliases

Enter the English location name and, when available, the French location name. Use the alias field for an alternate common name, short name, or legacy name. Use Add/Modify names in other languages when the location has names in additional languages. Each alternate-language row must include both a language and a name.

Type and coordinates

Select the location type that best describes the monitoring site. Enter latitude and longitude in decimal degrees using WGS84. Yukon and western Canada longitudes should normally be negative. The map button can be used to choose or verify coordinates; zoom in before saving map-selected coordinates.

Well association

If this location should be linked to an existing nearby well record, select Associate with nearby well, set an appropriate search radius, and choose the correct well from the nearby-well list. This is useful when a monitoring location represents or is directly tied to a borehole or well record.

Access and visibility

The Share with groups field controls which database roles can see the location. public_reader makes the record broadly visible to users with public read access. Use narrower groups for restricted records. Avoid combining public visibility with restricted intent.

The Publicly relevant checkbox marks locations that should be shown to the public. The Influenced by human activity checkbox flags locations affected by dams, mining, diversions, infrastructure, or other anthropogenic controls.

Vertical datum conversion

The datum fields describe how local or assumed elevation measurements relate to another vertical datum. For most locations, leave both datum fields as assumed datum and set the conversion to 0. Only enter a conversion when a reliable conversion is known and should be applied.

Networks and projects

Networks and projects are optional but recommended. They make locations easier to find, filter, report, and govern. Existing values can be selected from the lists. If a needed network or project is missing, type a new value and complete the add-new-network or add-new-project dialog.

Purpose, contact, and notes

Use contact details for the person, group, or organization responsible for the location. Use installation purpose for why the location was established, current purpose for why it is currently maintained, and notes for concise context that does not fit elsewhere.

Before saving

Check for duplicate names or codes, confirm coordinates on the map, confirm access groups, and verify networks, projects, and datum settings. After saving a new location, create any needed sub-locations before adding timeseries, samples, deployments, images, or documents that depend on the location.

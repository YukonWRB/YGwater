Add / modify sub-locations

Use this page to create or update sub-locations attached to a parent monitoring location. Sub-locations represent specific points within a location, such as a sampling point, sensor depth, bank, channel, intake, well screen, stilling well, or other repeated measurement position.

When to use a sub-location

Create a sub-location when multiple distinct measurement or sampling points belong to the same general monitoring location. This keeps the site grouped under one parent location while preserving the exact point used for samples, timeseries, deployments, images, or documents.

Add or modify mode

Choose Add new to create a sub-location. Choose Modify existing to select a sub-location from the table and edit its metadata. In modify mode, the associated parent location is locked so the sub-location is not accidentally moved to another location.

Associated location

Every sub-location must be associated with exactly one parent location. If the parent location is missing, create it first in Add / modify locations. Choose the parent carefully because downstream records may depend on this relationship.

Names

Enter a unique English sub-location name and, when available, a unique French name. Use clear names that distinguish the sub-location from other points at the same parent location, such as Main channel, Left bank, Well screen 1, or Staff gauge.

Coordinates

Enter latitude and longitude in decimal degrees using WGS84. Coordinates can be typed directly or chosen on the map. The map shows the parent location and helps verify that the sub-location is spatially reasonable.

Distance from parent location

The page warns when a sub-location is more than 10 meters from its parent location. This is not always wrong, but it should be intentional. If the distance warning appears, confirm that the selected parent location and coordinates are correct, then acknowledge the warning before saving.

Access groups

The Share with groups field controls which database roles can see the sub-location. public_reader is selected by default. Use restricted groups when the sub-location should not be broadly visible.

Notes

Use the sub-location note for concise details that help future users identify or interpret the point, such as marker descriptions, offset notes, sensor mounting details, or field conventions.

Before saving

Confirm the parent location, names, coordinates, access groups, and any distance warning. After saving, the sub-location can be used by timeseries, samples, deployments, and other records that need a more specific position than the parent location alone.
